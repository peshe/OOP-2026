## Функции от по-висок ред

Функция, някои формални параметри на която са функции, се нарича функция от по-висок ред.
Възможно е формален параметър на функция да е указател към функция, а също е възможно резултатът от
изпълнението на функция да е указател към функция. Това позволява да се реализират функции от по-висок ред.

Кодът на всяка функция на C++ се превежда до машинен код. Адресът на функцията наричаме адресът на първата инструкция във функцията.
Името на всяка функция може да се разглежда като константен указател към кода й. Стойността на указателя към функцията е адресът на нейния код.
Възможно е да се дефинират променливи, които са указатели към функции.

Примери:
```cpp
double (*p)(double, double); // дефинира променливa p от тип указател към функция от тип double с два аргумента също от тип double.
int (*q)(int, int*); // дефинира променлива q от тип указател къум функция от тип int с два параметъра единият от които е цял, а другия указател към int.
```
Нека са дефинирани следните функции за сортиране на числови редици:
```cpp
void bubbleSort(int, int*);
void mergeSort(int, int*);
void heapSort(int, int*);
```

Тогава, променливата r може да е указател към тези функции, ако е дефинирана така:
```cpp
void (*r)(int, int*);
```

Сега може да извършим сортиране, чрез променливата r.
Нека array e променлива от тип int*, т.е. масив от int-ове с дължина n. Нека също:
```cpp
r = mergeSort;
```
Тогава следните извиквания са еквивалентни:
```cpp
mergeSort(n, array); // директно обръщение към функцията mergeSort
r(n, array);         // индиректно обръщение към функцията mergeSort чрез r
```

# Enum
В C++ enum (enumeration) е дефиниран от потребителя тип, който позволява да се изброят символични имена за цели стойности. Използва се за представяне на малки множества от свързани константи, като така прави кода по-четим и по-лесен за поддръжка.

Ето един прост пример:
```cpp
#include <iostream>

enum Weather { Sunny, Cloudy, Rainy, Snowy };

int main()
{
    Weather today = Rainy;

    if (today == Rainy)
    {
        std::cout << "Don't forget your umbrella!" << std::endl;
    }
    else if (today == Sunny)
    {
        std::cout << "It's a great day for a walk!" << std::endl;
    }

    return 0;
}
```


## Задача 4:
Създайте програма, която използва изброим тип (enum) за представяне на четирите сезона: пролет, лято, есен и зима.

Изисквания:

1. Дефиниране на enum:
   Дефинирайте изброим тип (enum) с четири стойности – Spring (пролет), Summer (лято), Autumn (есен), Winter (зима).
   Може да използвате и българските имена, ако желаете.

2. Въвеждане на избор от потребителя:
   Програмата трябва да изведе инструкции, които да подканят потребителя да въведе число от 1 до 4, където:
* 1 означава Пролет
* 2 означава Лято
* 3 означава Есен
* 4 означава Зима

3. Преобразуване на избора към enum:
След въвеждане на числото, програмата трябва да преобразува избора към съответната стойност от enum-а.

4. Извеждане на съобщение:
Използвайки стойността на enum-а, програмата трябва да изведе подходящо съобщение, което описва типична дейност или характеристика за избрания сезон.
Примерни съобщения:
* Пролет: "Време е за разходки сред цъфтящите дървета."
* Лято: "Идеално време за плаж и море."
* Есен: "Листата падат и времето става по-хладно."
* Зима: "Можеш да караш ски или да се радваш на снега."

5. Проверка за валидност:
Ако потребителят въведе число извън диапазона 1-4, програмата трябва да изведе съобщение за невалиден избор.
(По желание) Допълнителни възможности:
Може да добавите функция, която приема enum стойност и връща подходящото съобщение, или да използвате масив от низове за съобщенията.

Примерен сценарий:

Въведете число от 1 до 4, за да изберете сезон:
1. Пролет
2. Лято
3. Есен
4. Зима

Вашият избор: 2

Изход: Лято: Идеално време за плаж и море.


## Задача 5

За всеки тип билет има определена цена:

- Стандартен: 20 лв.

- Пенсионерски: 12 лв.
- VIP: 50 лв.


Изисквания:
1. Дефинирайте изброим тип (enum class) с изброените видове билети. Например: (Standard, Student, Senior, VIP)

3. Напишете функция, която изчислява крайната сума за плащане при покупка на даден брой билети от избран тип.
4. Програмата трябва да показва меню с типовете билети и да подканва потребителя да избере тип чрез въвеждане на число (1-4).
